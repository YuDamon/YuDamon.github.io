可以看做是对Faster R-CNN进行“全卷积化”改进

Dai J, Li Y, He K, et al. R-FCN: Object Detection via Region-based Fully Convolutional Networks[J]. 2016.

知道这篇文章很久了，一直没读。今天终于看了

1 核心思想

Fast/Faster R-CNN网络结构被 RoI pooling 层分成了两部分：共享的全卷积子网（RoI pooling 层之前，提取全图feature），RoI-wise 的子网络（RoI pooling 层之后，对每个proposal分类，不共享；对一张有n个proposal的图，要计算n次，往往数百）。

对于图像级别的分类任务，是希望具有平移不变性(translation invariance)；而Detection，因为需要定位，所以一定程度上具有平移敏感性 (translation variance)。

在ResNet（除了最后一层都是全卷积的）的文章中，做Detection的时候，将 RoI pooling 层插入到两个子网络之间，就是为了让后面的层不再translation-invariant。这样之后，RoI-wise 的子网络更深了，提高了准确率但是计算自然慢了。

所以本文的核心思想就是将所有层的计算共享，即针对RoI pooling 层之后的子网络进行改进。

对于ResNet-101来说，将后面十层深的RoI-wise 的子网络全都共享了，所以最后会比Faster R-CNN快上2.5-20倍。准确率上也有不少提升(如何解释性能的提升？)

2 方法


如上图，用了一种特殊的卷积层作为前面全卷积网络的输出，来将 translation variance 引入到全卷积网络中，该层输出含待检测物体位置信息的（position-sensitive）的 score map。（“待检测物体位置信息” 确切地说是部位信息，比如物体网格的左上）

具体地说，为C个类别生成 k^2 个 position-sensitive 的 score map，总共 k^2 x (C+1) 通道的输出。 k^2 描述一个 k x k 的网格的空间信息（比如 k = 3，那么就分别对应 待检物体的 {top-left, top-center, top-right, …, bottom-right} 位置）

在最后一层，再接一个 position-sensitive 的RoI pooling 层，生成一个尺度为k^2 ， 通道数为(C+1) 的输出；然后Vote，最后完成分类。

另外一些细节：

  • Proposal也是RPN产生的，训练方式和 Faster R-CNN 的训练策略和步骤基本一样；
  • 也有Bounding Box 回归的子网；
  • 使用 Online Hard Example Mining (OHEM) ，几乎不会增加训练时间；
  • 使用了分割中用到的带孔卷积的trick（atrous），性能有提高。
