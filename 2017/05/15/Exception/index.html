2 C++

一些来自C++之父Bjarne Stroustrup的建议
参考链接

1- Don’t use exceptions where more local control structures will suffice; 局部控制能够处理时不要使用异常；
2- Throw an exception to indicate failure in a constructor; 构造函数中发生错误，通过抛出异常来指明；
3- Avoid throwing exceptions from destructors; 避免在析构函数中抛出异常；
4- Keep ordinary code and error-handling code separate; 保持普通程序代码和异常处理代码分开；
5- A library shouldn’t unilaterally terminate a program. Instead, throw an exception and let a caller decide; 编写的供别人调用的程序库，不应结束程序，而应通过抛出异常，让调用者决定如何处理（因为调用者必须要处理抛出的异常）；
6- Develop an error-handling strategy early in a design; 开发一个项目在设计阶段就要确定“错误处理的策略”；

2.1 语法实现


异常申明类似于函数形参的申明，可以是任何有效类型(当然包括C++中的类)的值，也可以是引用
比如

catch (const char* str) { }
catch (int) { }

特殊地，异常申明可以是省略号 … 此时可以捕获任何异常
和Python的一样，catch的顺序也很重要，… 应该放最后
