1 Switch-Case

python中没有现成的Switch-Case语句，

实现Switch Case需要被判断的变量是可哈希的和可比较的，这与Python倡导的灵活性有冲突。在实现上，优化不好做，可能到最后最差的情况汇编出来跟If Else组是一样的。所以Python没有支持。-
以上来自Python官方说法：PEP 3103 - A Switch/Case Statement.

可以用以下几种方案替代：
1 使用if-else
适用【case比较少，每个分支的操作较多较复杂】

2 使用dict类型

适用【case较多，每个分支的操作类型简单（不需要定义函数）】
或者再复杂一点点：

适用【case较多，每个分支的需要定义函数进行复杂操作】
但是很明显，使用dict类型也只能针对[类型一致/处理形式]的情况。举个栗子，函数参数个数就得一样，不能一个case的函数有参数一个没有。
所以呢，还是乖乖用if-else，所以情况都OK

3 脚本语言第一行

目的——指出这个文件中的代码用什么可执行程序去运行

`#!/usr/bin/Python `
告诉操作系统执行这个脚本的时候，调用/usr/bin下的python解释器；

`#!/usr/bin/env python `
这种用法是为了防止操作系统用户没有将python装在默认的/usr/bin路径里。当系统看到这一行的时候，首先会到env设置里查找python的安装路径，再调用对应路径下的解释器程序完成操作。

`#!/usr/bin/python ` 相当于写死了python路径;
`#!/usr/bin/env python`  会去环境设置寻找python目录,推荐这种写法
