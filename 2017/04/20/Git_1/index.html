（基础篇）一些Git的基本知识和操作

更多请参考 Pro Git 中文版

基础篇内容：
配置并初始化一个仓库（repository）、开始或停止跟踪（track）文件、暂存（stage）或提交（commit）更改。 配置 Git 来忽略指定文件模式、撤销错误操作、浏览项目的历史版本以及不同提交（commits）间的差异、远程仓库推送（push）以及从远程仓库拉取（pull）文件

1 初始化

初始化后，在本地代码库会自动创建一个 .git 隐藏目录

git init

所有 Git 需要的数据和资源都存放在这个目录中。但目前仅仅是按照既有的结构框架初始化好了里边所有的文件和目录，还没有开始跟踪管理项目中的任何一个文件

2 文件状态操作

工作目录下的每一个文件仅可能有”已跟踪或未跟踪“两种状态。
已跟踪文件：被纳入了版本控制的文件。在上一次快照中有记录，在工作一段时间后，状态可能处于未修改，已修改或已放入暂存区。 （初次clone某个仓库的时候，工作目录中的所有文件都属于已跟踪文件，并处于未修改状态）
未跟踪文件：既不存在于上次快照的记录中，也没有放入暂存区。

git status

2.1 一个例子

Step 1
先建一个新文件README

$ echo 'Hello' > README
$ git status
Untracked files: 
    README

此时，新的README未被跟踪。

Step 2
然后add一下README

$ git add README
$ git status
Changes to be committed: 
    new file:   README

add之后，README 文件：被跟踪，暂存状态

Step 3
若修改了一个已被跟踪的，的main.cpp

$ git status
Changes to be committed:
    new file:   README
Changes not staged for commit:
    modified:   main.cpp

此时，main.cpp状态是：已被跟踪，有修改，未暂存。
那么再用一次add暂存这个文件之后

$ git add main.cpp
$ git status
Changes to be committed:
    new file:   README
    modified:   main.cpp

现在两个文件都 已暂存

Step 4
若又一次修改了这个main.cpp，这时看下status

$ git status
Changes to be committed:
    new file:   README
    modified:   main.cpp

Changes not staged for commit:   
    modified:   main.cpp

发现会有暂存的，和还未暂存的两个版本的main.cpp（其实就是“又一次修改”前后的两个版本）
所以此时需要再次add

$ git add main.cpp
$ git status
Changes to be committed:
    new file:   README
    modified:   main.cpp

2.2 状态简览

git status -s

??：新添加的未跟踪文件
A：新添加到暂存区中的文件
左 M： 文件被修改并放入了暂存区
右 M ：文件被修改还未放入暂存区

2.3 add命令

可以是文件也可以是目录

git add <path>
git add .
git add *.cpp
git add index.html

2.4 忽略文件

若有无需纳入 Git 管理，也不希望总出现在未跟踪文件列表里的文件，可以创建 .gitignore 文件，列出要忽略的文件模式
一个例子来学习规范：

# 所有空行或者以 ＃ 开头的行都会被 Git 忽略
# 可以使用标准的 glob 模式匹配（这个什么意思 不懂）

# no .a files 
*.a

# 要忽略指定模式以外的文件或目录，可在前加（!）取反
# but do track lib.a, even though you're ignoring .a files above
!lib.a

# 匹配模式可以以（/）开头防止递归
# only ignore the TODO file in the current directory, not subdir/TODO
/TODO

# 匹配模式可以以（/）结尾指定目录
# ignore all files in the build/ directory
build/

# ignore doc/notes.txt, but not doc/server/arch.txt
doc/*.txt

# ignore all .pdf files in the doc/ directory
doc/**/*.pdf

注：GitHub 有十分详细的针对数十种项目及语言的 .gitignore 文件列表，https://github.com/github/gitignore

2.5 git diff

比较工作目录中当前文件和暂存区域快照之间的差异， 也就是修改之后还没有暂存起来的变化内容

git diff

若要查看已暂存的将要添加到下次提交里的内容

git diff --cached
git diff --staged （更高级版本中）

git diff 本身只显示尚未暂存的改动，而不是自上次提交以来所做的所有改动。
所以若使用git add .暂存了所有更新过的文件后，运行 git diff 后就会什么都没有

2.6 移除文件

要从 Git 中移除某个文件，就必须要从暂存区域移除，然后提交
Case 1
比如要删除README（Git不再跟踪，且从硬盘中删除）

$ rm README
$ git status
Changes not staged for commit:
    deleted:README

此时，README是未暂存状态
再运行 git rm 记录此次移除文件的操作

$ git rm README
rm 'README'
$ git status
Changes to be committed:
    deleted:README

此时，“移除README”被记录，README将不再纳入版本管理。如果删除之前修改过并且已经放到暂存区域的话，则必须要用强制删除选项 -f
Case 2
若想让文件保留在磁盘，但是并不想让 Git 继续跟踪

$ git rm --cached README

另外，mv同样可以用作文件重命名: 

git mv main.cpp main2.cpp

3 提交

先确定好文件是否都暂存了，然后git commit 提交（将索引内容添加到仓库中）

git commit -m "fix 3 bugs"

也可以跳过使用暂存区域 -a

git commit -a -m 'added new benchmarks'

git log

4 远程仓库推送push

4.1 添加

git remote add <shortname> <url>
git remote add origin git@github.com:YuDamon/Softmax.git
git remote add origin https://github.com/YuDamon/Softmax.git

此时用origin代替远程仓库git@github.com:YuDamon/Softmax.git

4.2 查看

git remote
git remote -v

git remote show [remote-name]
git remote show origin

4.3 推送

git push [remote-name] [branch-name]
git push -u origin master

使用-u之后，指定origin为默认主机，以后就可以不加任何参数使用git push了

4.4 移除和重命名

git remote rename originl oringin2
git remote rm origin2

5 从远程仓库拉取pull

git fetch [remote-name]
git pull [remote-name] [branch-name]

fetch：从远程获取最新版本到本地，不会自动merge
pull：是从远程获取最新版本，并merge到本地（相当于git fetch 和 git merge）
所以在实际使用中，git fetch更安全一些：在merge前，我们可以查看更新情况，然后再决定是否合并
